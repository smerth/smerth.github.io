5.1 What is the webpack-dev-server?

Webpack-dev-server uses a Node.js Express server. This uses the webpack-dev-middleware to serve a webpack bundle. Then it opens up a socket using Socket.IO that listens for changes and then those changes will be reloaded.

A lot of boiler plate for webpack projects includes the dev-server to make as an engineer much easier.

5.2 Setting up the webpack-dev-server

Install the webpack dev server

npm install webpack-dev-server --save-dev

Add a node to web pack.config.js to define the dev-server.

The dev server should have an attribute of inline true. The alternative to this would be false, in which case it would run the webpack dev server in an Iframe instead of in the actual window.

Set up our content base, which is where the content is coming from.

Specify which port we want to run it on, choose whatever feels right to you.

@ webpack.config.js

var webpack = require('webpack');
 
module.exports = {
    entry: './dist/app.js',
    output: {
        path: 'build',
        filename: 'bundle.js'
    },
    devServer: {
        inline: true,
        contentBase: './build',
        port: 3000
    },
    module: {
        loaders: [
            {
                test: /\.js$/,
                exclude: /(node_modules)/,
                loader: 'babel',
                query: {
                    presets: ['react', 'es2015']
                }
            },
            {
                test: /\.scss$/,
                loader: 'style-loader!css-loader!sass-loader'
            }, 
                {
                test: /\.(png|jpg)$/,
                loader: 'url-loader?limit=10000'
            }
     ]
    }
};

Create index.html file in the build folder

mkdir build && touch build/index.html

Edit index.html

@ index.html

<!DOCTYPE html>
<html>
<head>
    <title>Webpack Dev Server</title>
</head>
<body>
    <div id='react-container'></div>
    <script type="text/javascript" src="./bundle.js"></script>
</body>
</html>

Edit the package.json file.

Add scripts for build and start.

To use these scripts

npm run build

or

npm start

NPM comes with several predefined script names, one of which is "start". If you define your own you must precede it with "run" to execute it in the terminal.

@ package.json

{
  "name": "getting-started-webpack",
  "version": "1.0.0",
  "description": "Running a build with webpack",
  "main": "index.js",
  "scripts": {
    "build": "webpack",
    "start": "webpack-dev-server"
  },...

Finally we want to create inside of our dist folder a new file called app JS,

mkdir dist && touch dist/app.js

app.js is going to have all of our React content in it, therefore we'll need to require react and we also want to require ReactDOM and create a simple react component here.

Our react component is just going to be a class called app and we're going to render a div. The div is going to have an h1 and it's going to have a p element.

Finally, we want to mount this to our dom, so we'll use ReactDOM.render, we'll give it the name of the component which is app and then we want to say which container we want to mount this to, document.getElementByID with id = "react-container".

@ app.js

var React = require('react');
var ReactDOM = require('react-dom');
 
class App extends React.Component {
    render() {
        return <div>
            <h1>This is my AWESOME App</h1>
            <p>This is Eves app created with React, webpack and magic!</p>
        </div>
    }
}
 
ReactDOM.render(<App />, document.getElementById('react-container'));

Now that we have our component all set up, we should be ready to run webpack.

First, let's NPM run build,

npm run build

that's gonna create our bundle and

npm start

The app is running on local host 3,000. Check in the browser
