3.1 Load CSS with webpack

In addition to loading JavaScript with webpack, we can load CSS, Sass and LESS to style our pages. The benefit of loading CSS as a module is that webpack will only bundle the styles that our app uses.

We can also require or import styles for use with certain files. Also, it will perform transformations on Sass and LESS to turn it into CSS prior to loading into a browser.

3.1.1 Create a simple component using React JS.

Create a component. Create a Message Class extending React.Component. Give it a render function. Inside of the render function return a div. And inside of that div add an h1 to display this.props.title. Add a paragraph to display this.props.message.

Mount this component to the DOM using ReactDOM.render. Pass in the name of the component. Define the values of the properties in use in the component.

Use Document.getElementById with an id of "react-container" to target an insertion point for this component in the DOM.

@ main.js

var React = require('react');
var ReactDOM = require('react-dom');
require('./style.css');
 
class Message extends React.Component {
    render() {
        return (<div>
                    <h1>{this.props.title}</h1>
                    <p>{this.props.message}</p>
                </div>);
    }
}
 
ReactDOM.render(<Message title="Email Joe" message="Can you email him?"/>, 
    document.getElementById('react-container'));

Install React and React-dom. Save to dependancies (not dev dependancies,) since both libraries are needed on the front end

npm install react react-dom --save

Create the element we are targeting with our React component.

@index.htm

<!DOCTYPE html>
<html>
<head>
    <title>React &amp; Webpack</title>
</head>
<body>
    <div id='react-container'></div>
    <script type="text/javascript" src="./build/bundle.js"></script>
</body>
</html>

Run webpack.

webpack

The React component being mounted. We see the index.html extension here showing us that this is in fact the correct folder. Everything is working okay.

3.1.2 Add styles

When we load CSS with webpack it will be made part of that bundle and will only use the styles that we need. Create style.css.

@ terminal

touch style.css

Edit styles

@ style.css

body {
    background-color: black;
}
div {
    background-color: green;
    color: white;
}

3.1.3 Add another loader for CSS

Add another loader inside of the loaders array. It will be another object with a key of test.

This time we're going to use a CSS extension to find that file.

The regex /\.css$/ matches the css extension.

We will use the style loader and pipe in the CSS loader.

@ webpack.config.js

module.exports = {
    entry: './src/main.js',
    output: {
        path: 'build',
        filename: 'bundle.js'
    },
    module: {
        loaders: [
            {
                test: /\.js$/,
                exclude: /(node_modules)/,
                loader: 'babel',
                query: {
                    presets: ['react', 'es2015']
                }
            },
            {
                test: /\.css$/,
                loader: 'style-loader!css-loader'
            }
        ]
    }
};

Install the style loader and CSS loader as dependencies.

@ terminal

npm install style-loader css-loader --save-dev

Lastly, require the CSS in the JavaScript file. Use require() to pull in the style.css file.

@ main.js

require('./style.css');

We will run webpack again.

And now if we take a look at this in our browser, we should notice that style has been applied.

3.2 Transpiling Sass with webpack

Webpack also has a loader for Sass, the CSS extension language that allows you to write more complex CSS code using variables, mixins, and nesting.

Let's change our our style sheet to a style.scss file by renaming it.

@ terminal

mv style.css style.scss

And then we are going to use some SCSS

@ style.scss

$green: #00FF00;
$white: #FFFFFF;
 
div {
    background-color: $green;
    color: $white;
}

Edit main JS file where we'll use SCSS instead of CSS as the file extension of the file that we want to import.

@ main.js

require('./style.scss');

Now, over in our terminal window we need to install the sass and node-sass dependencies.

@ terminal

npm install sass-loader node-sass --save-dev

Edit webpack.config.js to load SCSS files and then we are going to chain on a sass-loader.

@ webpack.config.js

module.exports = {
    entry: './src/main.js',
    output: {
        path: 'build',
        filename: 'bundle.js'
    },
    module: {
        loaders: [
            {
                test: /\.js$/,
                exclude: /(node_modules)/,
                loader: 'babel',
                query: {
                    presets: ['react', 'es2015']
                }
            },
            {
                test: /\.scss$/,
                loader: 'style-loader!css-loader!sass-loader'
            }
     ]
    }
};

Run webpack we should see that bundled correctly and if we reload, we notice that that change has taken place. SCSS has been preprocessed into CSS and it runs in the browser.

Webpack also has a loader for LESS if that's your CSS framework of choice.

3.3 Loading images with webpack

The process of loading images with webpack is very similar to loading CSS. With webpack we load images by using URL loader. Webpack "in-lines" a URL to the image bundle and then returns it from require.

"in-lining" images will reduce the number of HTTP requests which will speed up our applications a lot.

First install url-loader and file-loader to our dev dependencies.

@ terminal

npm install url-loader file-loader --save-dev

Edit style.scss file to change the div so that we have a background property of URL and then we'll pass in the name of our URL file, our image file, which is "rubicon.jpg".

@ style.scss

div {
    background: url(rubicon.jpg);
    height: 1000px;
}

Edit webpack.config.js add an image loader.

Test for pngs and jpgs using the regex /\.(png|jpg)$/.

Add is the URL loader. You can give it a limit of 20 thousand if necessary (if an image is bigger than 20 kilobytes, it's going to create a direct URL to the image asset. Remember that file loader that we installed before? That's gonna take care of loading the image in the event that it exceeds the size limit that we enforced here.)

@ webpack.config.js

module.exports = {
    entry: './src/main.js',
    output: {
        path: 'build',
        filename: 'bundle.js'
    },
    module: {
        loaders: [
            {
                test: /\.js$/,
                exclude: /(node_modules)/,
                loader: 'babel',
                query: {
                    presets: ['react', 'es2015']
                }
            },
            {
                test: /\.scss$/,
                loader: 'style-loader!css-loader!sass-loader'
            },
            {
                test: /\.(png|jpg)$/,
                loader: 'url'
            }
     ]
    }
};

Over in our terminal, run webpack.

@ terminal

webpack

The image is being loaded.

In developer tools see that this div now has a background property, using the data image URI that's been created by webpack automatically just by using the URL loader.
