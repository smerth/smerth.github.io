4.1 Working with arguments

Arguments are specificied by the user when your script is run. They can be almost anything, a file or folder, a user name, a string of text you want to search for etc.

They are passed into the script by typing them after the script name.

4.1.0.1 4.1.1.sh

#! /bin/bash
 
# arugments are assigned numbered variables according to their position on the command line.
echo $1

This program echos the value of the first argument provided on the command line

4.1.0.2 4.1.2.sh

To deal with an array of arguments of arbitrary length bash gives us an array of arguments $@ also bash gives us $# which is the number of items in the arguments array.

#! /bin/bash
 
for i in $@
do
    echo $i
done
 
echo "There were $# arguments."

4.2 Working with flags

You know you can feed flags into your C programs on the command line using the -. You can do the same with a Bash script.

4.2.0.1 4.2.1.sh

#! /bin/bash
 
while getopts u:p: option; do
    case $option in
        u) user=$OPTARG;;
        p) pass=$OPTARG;;
    esac
done
 
echo "User: $user / Pass: $pass"

Run by passing in the arguments -u your name -p yourpassword

In the above case we specify two flags with each one followed by a collon. This means we expect to use them and they are added to the $options array.

However if we add flags and do not follow them with a collon we are only able to see if they were entered on the command line when the script is run.

4.2.0.2 4.2.2.sh

#! /bin/bash
 
while getopts u:p:ab option; do
    case $option in
        u) user=$OPTARG;;
        p) pass=$OPTARG;;
        a) I got the A flag.
        b) I got the B flag.
    esac
done
 
echo "User: $user / Pass: $pass"

run

bash 4.2.2.sh -u Apple -p Orange -a -b

adding a colon at the beginning of the list of flags lets bash know you wan to know about any flag that that is entered when the program is run but are not defined in the program.

4.2.0.3 4.2.3.sh

#! /bin/bash
 
# added a colon before the flag list
while getopts :u:p:ab option; do
    case $option in
        u) user=$OPTARG;;
        p) pass=$OPTARG;;
        a) echo "Got the A flag";;
        b) echo "Got the B flag";;
        ?) echo "I don't know what $OPTARG is...";;
    esac
done
 
echo "User: $user / Pass: $pass"

run

bash 4.2.3.sh -u Sampson -p 54321 -a -b -f example

4.3 Getting input during execution

4.3.0.1 4.3.1.sh

#! /bin/bash
 
echo "What is your good name?"
read name
 
# -s stands for silent the input won't be shown in terminal
echo "What is your preferred password?"
read -s pass
 
# you can write everything on the same line with -p for prompt
read -p "What's your favorite type of camel?" camel
 
echo name: $name, pass: $pass, camel: $camel

4.3.0.2 4.3.2.sh

Getting input in a handy menu form:

#! /bin/bash
 
# "select" is followed by the "variable name" you want to store the selection in folloed by the word "in"
select animal in "cat" "dog" "bird" "cucumber"
 
do
    echo "You selected $animal!"
    #remember to break out of the loop
    break
done

4.3.0.3 4.3.3.sh

Use the select list with a case function

#! /bin/bash
 
# "select" is followed by the "variable name" you want to store the selection in folloed by the word "in"
select option in "cat" "dog" "bird" "cucumber" "quit"
 
do
    case $option in
        cat) echo "But cats are selfish and loathsome.";;
        dog) echo "Arff... Your answer is consistant with above average intelligence.";;
        bird) echo "We have tweeted your response to your employer!";;
        cucumber) echo "Ummmm....";;
        quit) break;;
        *) echo "I'm not sure what that input means,... unless... maybe you are a HAACCKKKER!!!"
    esac
done

4.4 Ensuring a response

4.4.0.1 4.4.1.sh

Here we use an if/else statement to prevent the program from executing if it doesn't get the required number of arguments.

#! /bin/bash
 
if [ $# -lt 3 ]; then
    cat <<- EOM
    This command requires 3 arguments:
    username, userid, and favorite number.
    EOM
else
    # The program goes here
    echo "Username: $1"
    echo "UserID: $2"
    echo "Favorite Number: $3"
fi

4.4.0.2 4.4.2.sh

Here we use the -z option to check if the variable a is empty

#! /bin/bash
 
read -p "Favorite animal? " a
while [[ -z "$a" ]]; do
    read -p "I need an answer! " a
done
echo "$a was selected."

4.4.0.3 4.4.3.sh

Assume a default answer if none is given

#! /bin/bash
 
read -p "Favorite animal? [cat] " a
while [[ -z "$a" ]]; do
    a="cat"
done
echo "$a was selected."

4.4.0.4 4.4.4.sh

Basic validation of input using a regex.

#! /bin/bash
 
read -p "What year? [nnnn] " a
while [[ ! $a=~ [0-9]{4} ]]; do
    read -p "A year, please! [nnnn] " a
done
echo "Selected year: $a"
